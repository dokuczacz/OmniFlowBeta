## Getting a DOI via Zenodo

To mint a DOI for this repository:

1. **Connect GitHub to Zenodo**:
   - Visit https://zenodo.org/
   - Log in with your GitHub account
   - Go to https://zenodo.org/account/settings/github/
   - Find `dokuczacz/OmniFlowBeta` and toggle it ON

2. **Create a GitHub Release**:
   - Go to https://github.com/dokuczacz/OmniFlowBeta/releases
   - Click "Create a new release"
   - Tag version: `v0.1.0`
   - Release title: `OmniFlow Beta v0.1.0`
   - Add release notes
   - Publish release

3. **Get Your DOI**:
   - Zenodo automatically archives the release
   - Visit https://zenodo.org/account/settings/github/
   - Click on the newly archived version
   - Copy the DOI badge and citation text

4. **Update Citation Files**:
   - Replace `10.5281/zenodo.XXXXXXX` with your actual DOI in:
     - This file (README_citation_snippet.md)
     - CITATION.cff
     - Main README.md (if DOI badge is added)

## Citation.cff File

This repository includes a `CITATION.cff` file for automated citation metadata. GitHub automatically detects this file and provides a "Cite this repository" button with formatted citations.

**Learn more**: https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-citation-files

## Why Cite?

Citing software helps:
- Give credit to developers and maintainers
- Track the impact and usage of open-source projects
- Improve reproducibility in research
- Support funding and sustainability efforts
